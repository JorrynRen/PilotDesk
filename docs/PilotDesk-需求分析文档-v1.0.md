## 1. 项目概述

### 1.1 项目背景

随着 AI Agent 技术的快速发展，开发者需要一个统一的桌面客户端来管理和使用多种 AI Agent（如 Claude Code、Hermes Agent 等）。现有的 Web 端解决方案存在以下痛点：

- 多个 Agent 平台分散，频繁切换效率低下
- 缺乏本地化的会话管理和灵感沉淀能力
- API 密钥管理不安全，依赖浏览器存储
- 无法与本地开发工作流深度集成

### 1.2 项目目标

构建一个 **Windows 原生桌面应用**，作为多 AI Agent 的统一客户端，提供会话管理、灵感系统、API 配置等核心功能。

### 1.3 目标用户

| 用户类型 | 使用场景 | 核心需求 |
|---------|---------|---------|
| 独立开发者 | 日常编码辅助 | 快速切换 Agent、管理会话历史 |
| AI 研究者 | 多模型对比测试 | 多 API 配置、快速切换 |
| 内容创作者 | 灵感管理 + AI 辅助写作 | 灵感收集、快速调用 AI |
| 团队技术负责人 | 统一团队 AI 工具 | 标准化 Agent 配置 |

## 2. 功能需求

### 2.1 核心功能（P0）

#### 2.1.1 多 Agent 会话管理
- 支持创建、删除、重命名会话
- 会话列表展示，支持会话间快速切换
- 会话模式切换（支持 Claude Code / Hermes Agent 等多种 Agent 类型）
- 消息气泡展示（用户消息靠右，Agent/系统消息靠左）
- SSE 流式输出支持

#### 2.1.2 灵感系统
- 灵感面板：展示所有灵感卡片
- 灵感创建：支持标题、内容、图标选择（24+图标）
- 灵感搜索：本地实时过滤
- 灵感应用：点击灵感自动填入会话输入框
- 灵感市集：浏览和管理灵感集合
- 从消息中收藏灵感

#### 2.1.3 API 配置管理
- 支持多 API 提供商配置（Anthropic、OpenAI 格式）
- 每个配置包含：名称、API Key、API 端点(URL)、模型标识
- 配置列表展示，支持新增、编辑、删除
- **拖拽排序**调整配置优先级
- **测试连接**功能验证 API 可用性

#### 2.1.4 设置系统
- **通用设置**：工作区目录（原生目录浏览选择）、默认 Agent 等
- **Agent 参数配置**：同时支持 Claude Code 和 Hermes Agent 的参数设置
  - Claude Code：URL(API端点)、Model、Max Tokens、Temperature 等
  - Hermes Agent：对应参数配置
- **API 配置**：列表式管理（上述2.1.3）
- **关于页面**：版权信息、MIT 协议、版本号

### 2.2 重要功能（P1）

#### 2.2.1 右侧面板
- 灵感面板：展示、搜索、新建灵感
- 技能面板：内置技能 + 自定义技能（localStorage 持久化）
- 面板可折叠/展开

#### 2.2.2 消息编辑
- 支持编辑已发送的用户消息（填充回输入框）

#### 2.2.3 WebSocket 通信
- 与 Node.js Sidecar 通过 WebSocket 通信
- 连接状态实时显示（状态栏）

### 2.3 辅助功能（P2）

- 多会话模式（下拉向上展开）
- 窗口控制：最小化、最大化/还原、关闭
- 标题栏双击最大化/还原
- 整个标题栏可拖动
- 主题色支持 CSS 变量

## 3. 非功能需求

### 3.1 性能要求
- 应用启动时间 < 3 秒
- UI 渲染帧率 ≥ 60fps
- SSE 流式输出延迟 < 500ms

### 3.2 安全要求
- API Key 存储于 localStorage（前端方案）
- 不在日志中输出敏感信息
- MIT 开源协议

### 3.3 兼容性
- 目标平台：Windows 10/11
- 系统要求：64位，WebView2 运行时

## 4. 技术选型

| 层级 | 技术栈 | 说明 |
|------|--------|------|
| 前端 UI | React 19 + TypeScript + TailwindCSS | 现代化响应式界面 |
| 状态管理 | Zustand | 轻量级状态管理 |
| 桌面框架 | Tauri 2.0 | 轻量级原生桌面应用 |
| 后端 | Rust (rusqlite) | 本地数据持久化 |
| 通信层 | Node.js Sidecar (WebSocket) | Agent 通信桥接 |
| 拖拽排序 | @dnd-kit/sortable | API 配置拖拽 |
| 原生对话框 | @tauri-apps/plugin-dialog | 目录选择器 |
